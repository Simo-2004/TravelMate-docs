# 3.3.1 Usability

Usability requirements define how the system must present itself to the Traveler and how readily its functions must be reached.

**NFR-U.1 — Interface conventions.** The application shall follow the interface conventions of its target mobile platform, so that navigation, controls, and gestures behave as users already expect.

**NFR-U.2 — Layout adaptability.** The interface shall adapt to the range of screen sizes of the supported devices without loss of content and without horizontal scrolling.

**NFR-U.3 — Depth of navigation.** Every principal function — searching, saving an item, editing one's profile, opening a conversation — shall be reachable within three interactions from the main navigation.

**NFR-U.4 — Feedback on actions.** Every action that alters stored data shall produce an immediate, visible confirmation stating its outcome.

**NFR-U.5 — Registration effort.** A new Traveler shall complete registration within a single form, with each invalid field reported individually so that a rejected submission need not be composed again from the beginning.

**NFR-U.6 — Text scaling.** The interface shall respect the text-size preference set at operating-system level, within bounds that preserve the integrity of the layout.

## Deferred to future releases

**Theming.** A dark presentation offered alongside the light one.

**Guided onboarding.** A multi-step introduction presenting the platform's functions to a Traveler using it for the first time.

**Accessibility conformance.** Formal compliance with WCAG 2.1 Level AA, including audited contrast ratios and verified screen-reader semantics.
