# 1.4 Definitions, Acronyms & Abbreviations

- **RAD** — Requirements Analysis Document, this document
- **SDD** — System Design Document, which specifies how the system is built
- **ODD** — Object Design Document, which specifies the classes realising the design
- **EBC** — Entity–Boundary–Control, the pattern by which the analysis objects in [3.4.3](../proposed-system/system-models/object-model) are classified
- **UML** — Unified Modeling Language, the notation of the diagrams in [3.4](../proposed-system/system-models/)
- **Traveler** — the actor using the application; see [3.4.3.1](../proposed-system/system-models/object-model)
- **Companion** — a potential travel mate presented by the system
- **Bookmark** — a saved reference to a trip or a companion
- **Auto-reply** — the response a companion gives in conversation, determined by the system from the content of the message

Domain and business terminology is defined in [4. Glossary](../glossary). Terms belonging to the design — the storage mechanism, the cryptographic algorithms, and the components realising them — are defined in the System Design Document, where those choices are made.
